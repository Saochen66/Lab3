# Lab1 白盒测试（calcShortestPath）

## 1. 方法与原则

被测函数：`String calcShortestPath(String word1, String word2)`。

白盒测试采用控制流/基本路径法，不采用等价类划分。

## 2. 基本路径识别

依据当前实现提取 7 条核心基本路径：

| 路径编号 | 路径描述 | 期望输出 |
|---|---|---|
| Path1 | 起点不存在，直接返回 | `No "word1" in the graph!` |
| Path2 | 单目标分支中目标不存在 | `No "word2" in the graph!` |
| Path3 | 单目标分支，目标存在但不可达 | `No path from 'w1' to 'w2'!` |
| Path4 | `word2` 为空，遍历后无可达节点 | `No reachable nodes from 'w1'.` |
| Path5 | `word2` 为空，遍历后有可达节点 | 含 `a->b (length=1)` |
| Path6 | 单目标分支且可达 | `a->b->c (length=2)` |
| Path7 | `word2` 为空，遍历中出现不可达目标并追加 No path 行 | 包含 `No path from 'a' to 'c'.` |

## 3. 白盒测试用例（分文件）

| 用例编号 | 初始图边集合 | 调用 | 期望输出 | 覆盖路径 |
|---|---|---|---|---|
| TC1 | `a->b` | `calcShortestPath("x","b")` | `No "x" in the graph!` | Path1 |
| TC2 | `a->b` | `calcShortestPath("a","c")` | `No "c" in the graph!` | Path2 |
| TC3 | `a->b, c->d` | `calcShortestPath("a","c")` | `No path from 'a' to 'c'!` | Path3 |
| TC4 | `a->a` | `calcShortestPath("a","")` | `No reachable nodes from 'a'.` | Path4 |
| TC5 | `a->b` | `calcShortestPath("a","")` | 结果包含 `a->b (length=1)` | Path5 |
| TC6 | `a->b, b->c` | `calcShortestPath("a","c")` | `a->b->c (length=2)` | Path6 |
| TC7 | `a->b, c->c` | `calcShortestPath("a","")` | 同时包含 `a->b (length=1)` 与 `No path from 'a' to 'c'.` | Path7 |
